The Layered Prompting Method

The single biggest mistake students make is writing one large prompt asking Copilot to build everything at once. The result is always a generic, unstyled page that barely resembles their design. The layered method breaks the build into four distinct stages — each building on the last — and consistently produces results that are far closer to the original Figma mockup.

Before you start each layer: open your Figma file. Have your Assets page (Page 04) visible. Every value you prompt Copilot with should come directly from your documented design — not from memory.

Layer 1 — Structure

Build the HTML skeleton — no styling yet

Describe every section of your homepage in order from top to bottom. Be specific about what each section contains. Do not mention colours, fonts, or sizing in this prompt — that comes in Layer 2. Use semantic HTML5 elements.

What to include in your Layer 1 prompt

✓

The page title and navigation structure (logo position, link labels)

✓

Every section in order: hero, features, about, gallery, testimonials, contact, footer

✓

What each section contains: heading, subheading, image, button, card grid, form, etc.

✓

The number of columns in any grid layout

✓

Explicit instruction: semantic HTML5, no CSS

Layer 1 — Prompt Template

layer-1-prompt.txt
"Build the HTML structure for a [your category] website homepage.
Use semantic HTML5 elements. Do not add any CSS yet.
The page should have these sections in this order:
1. Navigation bar: logo on the left named '[Your Site Name]',
 links on the right: [Link 1], [Link 2], [Link 3], [Link 4].
2. Hero section: a large heading '[Your headline]',
 a subheading '[Your subheading]', and a button '[CTA text]'.
3. [Section name]: [describe contents — e.g. three cards each with
 an image, a heading, and a short paragraph].
4. [Continue for each section...]
5. Footer: [describe footer contents]."

After Layer 1: review the HTML structure in the browser. Does every section exist? Is the order correct? Fix any structural issues before moving to Layer 2 — styling a broken structure makes problems harder to fix, not easier.

Layer 2 — Styling

Apply your colour system, typography, and spacing

Open your Figma Assets page (Page 04). Every value in this prompt should come directly from your documented design. Copy HEX codes exactly. Use the exact Google Font name. Reference your spacing scale. Do not guess — if you documented it, use it.

What to include in your Layer 2 prompt

✓

Exact HEX codes for every colour in your palette — label each one (primary, accent, background, text)

✓

Font names exactly as they appear on Google Fonts, with weights and sizes for each hierarchy level

✓

Line-height values for body text and headings

✓

Section padding (vertical), container max-width, card padding, and gap values

✓

Button styles: background colour, text colour, border-radius, padding

✓

Navigation bar styling: background, link colour, height

Layer 2 — Prompt Template

layer-2-prompt.txt
"Now add CSS styling to the HTML structure. Use these exact values:
COLOURS:
 Background: #[your HEX]
 Primary text: #[your HEX]
 Accent / brand colour: #[your HEX]
 Button background: #[your HEX] Button text: #[your HEX]
 Card background: #[your HEX] Footer background: #[your HEX]
TYPOGRAPHY (import from Google Fonts):
 Headings: [Font Name] — H1: [size]px bold, H2: [size]px bold, H3: [size]px semibold
 Body: [Font Name] — [size]px, line-height [value]
 Navigation links: [size]px, [weight], [any letter-spacing]
 Button label: [size]px [weight]
SPACING AND LAYOUT:
 Content container max-width: [value]px, centred with auto margins
 Section vertical padding: [value]px top and bottom
 Card padding: [value]px Card gap: [value]px
 Navigation bar height: [value]px
BUTTONS: border-radius [value]px, padding [v]px [h]px"

After Layer 2: open your Figma mockup side-by-side with the browser. Compare colours, font sizes, and spacing section by section. Note every discrepancy — you will fix these in Layer 4.

Layer 3 — Responsive & Accessible

Add mobile breakpoints, collapsing behaviour, and accessibility basics

Refer to your Figma mobile frames. For every section on desktop, you should have already designed how it behaves on mobile. Describe each change explicitly — Copilot needs to know specifically what changes at each breakpoint. This layer also covers accessibility — Week 11's WCAG requirements and Assessment 2's Responsiveness and Accessibility criterion are addressed together here, since they're assessed together.

What to include in your Layer 3 prompt

✓

Breakpoint values (e.g. tablet below 768px, mobile below 480px)

✓

What happens to the navigation at each breakpoint (hamburger menu, stacked links)

✓

What happens to any grid or multi-column layout (e.g. three columns becomes one)

✓

What happens to font sizes at mobile width

✓

Colour contrast requirements (WCAG AA, 4.5:1 for body text)

✓

Descriptive alt text on every image

✓

Semantic HTML landmarks and keyboard-navigable interactive elements

Layer 3 — Prompt Template

layer-3-prompt.txt
"Make this site responsive and accessible.

BREAKPOINTS:
 Tablet: below [768]px
 Mobile: below [480]px

NAVIGATION:
 Below [768]px: replace the horizontal nav links with a hamburger
 menu icon that toggles a full-width dropdown menu. Menu must be
 operable by keyboard (Tab and Enter).

LAYOUT:
 [Section name] grid: [3] columns on desktop becomes [1] column
 below [480]px. Cards stack vertically with [16]px gap.
 [Repeat for each multi-column section...]

TYPOGRAPHY:
 H1 scales from [size]px desktop to [size]px mobile.
 Body text remains at minimum 16px on all screen sizes.

ACCESSIBILITY:
 Ensure all body text meets a 4.5:1 contrast ratio against its
 background (WCAG AA). Add descriptive alt text to every image —
 describe what the image shows, not the filename. Use semantic
 landmarks: <nav>, <main>, <header>, <footer>. All interactive
 elements (links, buttons, menu) must be reachable and usable via
 keyboard alone."

After Layer 3: test at 1440px and 375px. Tab through the page using only your keyboard — can you reach every link and button? Check contrast using a tool like WebAIM's contrast checker.

Layer 4 — Refinement

Close the gap between the output and your Figma design

Layer 4 is the most important layer for your grade. This is where you demonstrate critical evaluation — identifying specific gaps between your Figma design and the built output, and prompting precisely to fix them. Vague refinement prompts produce vague results. Be specific.

How to write an effective refinement prompt

✓

Identify one specific problem at a time. Do not batch multiple issues into one prompt.

✓

Describe what IS there, what SHOULD be there, and where it is on the page.

✓

Reference your Figma design explicitly — "In my Figma mockup the hero has..."

✓

Include the exact value you want — colour HEX, pixel size, percentage.

Layer 4 — Refinement Prompt Examples

layer-4-examples.txt
# Fixing a colour problem:
"The hero section has a white background.
 In my Figma design it should have a full-width background image
 with a dark overlay. Add a dark overlay at 60% opacity (#000000)
 over the hero image so the white heading text is clearly readable."

# Fixing a spacing problem:
"The gap between the three feature cards is too large.
 In my Figma design the gap between cards is 16px not 32px.
 Reduce the gap between cards in the card grid to 16px."

# Fixing a typography problem:
"The navigation links are displaying in the body font (Inter).
 In my Figma design the nav links should be in uppercase with
 2px letter spacing. Add text-transform: uppercase and
 letter-spacing: 2px to the navigation link CSS."

# Fixing a layout problem:
"The about section currently has the image on the left and text on
 the right. In my Figma design it is the opposite — text on the
 left, image on the right. Please reverse the order."
The Comparison Method

Review every layer the same way

Use this structured approach every time you review a layer. Open your Figma mockup and the browser side by side (50/50 split on your screen) and work through these questions section by section.

What to look for and how to fix it
Colours match?
Compare every background and text colour. If wrong, copy the HEX from Figma and re-prompt Copilot with the exact value — do not describe it as a colour name.
Fonts correct?
Check family, size, weight, and letter-spacing. Open DevTools (F12) to inspect the computed font-size and family, and compare against your Figma text styles.
Spacing correct?
Check section padding, card gap, and element margins. Measure in Figma using the inspect panel, and compare pixel values directly against the live site.
Layout matches?
Check column counts, alignment, and element order. Compare desktop and mobile views side by side against your Figma frames.
Common Failures — and How to Fix Them

When Copilot goes wrong

Problem
Fix
Copilot ignores part of my prompt
Your prompt is too long. Split it into smaller, focused prompts — one section or one concern at a time.
The output looks nothing like my design
You described the content but not the style. Return to Layer 2 and add your exact HEX codes, font names, and pixel values.
The font is not loading
Make sure you named the Google Font exactly. Check fonts.google.com — "Playfair Display" not "Playfair".
Mobile layout is broken
Layer 3 was too vague. Return and describe each section's mobile behaviour explicitly — do not say "make it responsive", say exactly what changes.
The hero image is not showing
Copilot uses placeholder images. Replace the src attribute with your actual image file path.
Colours are close but not right
You used colour names instead of HEX codes. Always use exact HEX values: #1A1A1A not "dark grey".
Cards are not equal height
The card container needs display: flex and the cards need flex: 1. Prompt this specifically.
Copilot keeps adding things I did not ask for
Start each prompt with "Only change [specific thing]. Do not modify anything else on the page."
Copilot ignores my accessibility instructions
You buried it in a long prompt. Ask for accessibility fixes as their own focused prompt, the same way you would for a colour or spacing fix.
My contrast checker is failing
You used a colour combination that looks fine to you but fails WCAG AA. Check every text/background pairing with a contrast tool and adjust the HEX values — do not guess.
My site is not keyboard-navigable
Interactive elements are probably styled <div>s instead of real <button>/<a> tags — divs are not keyboard-focusable by default. Ask Copilot to convert them to semantic, focusable elements.

For Your Presentation

Show the process, not just the result

Your presentation should walk through your prompting strategy layer by layer. Show a prompt, show the output, explain what you did next.

Be honest about what failed

The most impressive presentations are honest about failures. "This prompt produced this — I then refined it by..." shows genuine critical thinking.

Have your Figma mockup open

Switch between your Figma design and your live site to show the comparison. The gap you closed is the evidence of your skill.

Prepare for follow-up questions

The lecturer will ask you to explain a specific decision. Know why you wrote each key prompt and what you would do differently.

The goal is not a perfect website. The goal is a website you can explain, defend, and reflect on critically. A site that is 80% faithful to your Figma design — with a clear, honest account of how you got there — will always outperform a polished site you cannot explain.
