Create your GitHub account

Open your browser and go to github.com

Click the green Sign up button in the top right corner

Enter your email, create a password, and choose a username — keep it simple (e.g. johnsmith). Your username becomes part of your website URL.

Verify your account by clicking the link sent to your email inbox

Tip: Use your university email address so you can access the GitHub Education free plan, which gives you extra features including free private repositories and GitHub Copilot.

Create a repository for your website

Once logged in, click the + icon in the top right corner, then choose New repository

In the Repository name field type: yourusername.github.io

Replace yourusername with your actual GitHub username — e.g. if your username is johnsmith, name it johnsmith.github.io

Set the repository to Public (not Private)

Leave all other settings as default and click Create repository

Important: The repository name must match exactly — yourusername.github.io. One wrong character and GitHub Pages will not work. This is the most common mistake!

Upload your website files

Open your new repository — you will see a message saying it is empty. That is expected.

Click uploading an existing file (shown as a link in the empty repo page)

Drag and drop all your website files — index.html, style.css, and any image or JavaScript files

Scroll to the bottom and click the green Commit changes button

Important: Your main page file must be named index.html — GitHub Pages looks for this exact filename first. If your file has a different name, rename it before uploading.

✓

Step 3 complete

Files uploaded and committed — one step from going live.

Step 4 of 5
0 / 4 complete

Turn on GitHub Pages

At the top of your repository, click the Settings tab — you may need to scroll right to see it

In the left sidebar scroll down and click Pages under the "Code and automation" section

Under Branch, click the dropdown that says "None" and select main — leave the folder set to / (root)

Click Save

Your site will go live in about 1–2 minutes. The page will show a box with your URL — yourusername.github.io — share it with anyone and they can view your site from any device, anywhere, for free.

✓

Step 4 complete

GitHub Pages is on — your site will be live in 1–2 minutes.

Updating your website in the future

No need to start over — GitHub replaces files automatically when you upload the same filename.

Log in to GitHub, click your profile picture → Your repositories → click your website repository

Click Add file → Upload files and drag in your updated file (e.g. your new index.html). GitHub automatically replaces the old version.

Scroll down and click Commit changes — your live site updates in 1–2 minutes

Watch out

Common mistakes to avoid

These are the four things that trip students up most often — double-check each one if your site isn't loading.

Mistake 01

Wrong repository name

The repo name must be exactly yourusername.github.io — matching your GitHub username precisely. One wrong character and Pages won't activate.

Mistake 02

Wrong main filename

Your homepage must be called index.html — not homepage.html, page1.html, or anything else. GitHub Pages loads index.html first.

Mistake 03

Repository set to Private

GitHub Pages only works on Public repositories on the free plan. If your site won't load, check Settings and make sure Visibility is set to Public.

Mistake 04

Branch not set to main

If your site disappears after an update, go back to Settings → Pages and confirm the Branch dropdown still shows main, then click Save again.
