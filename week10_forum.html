01

THE COMPARISON
METHOD

Figma and Browser,
Side by Side.

Figma

Source of Truth

  • Your hi-fi mockup — the reference
  • Zoom to 100%
  • Work one section at a time — hero first
  • Note exact HEX, font, and spacing values
  • Screenshot before prompting
VS
Browser

Live Output

  • Your live GitHub Pages site
  • Open DevTools (F12)
  • Look at the same section as Figma
  • List specific differences
  • Each difference becomes a Layer 2 fix

The comparison method applies to every layer. It is never one prompt and done. It is always: prompt → compare → identify → fix.

LAYER 2
STYLING

Layer 2 / 01

Colour

Pulled from your Figma palette. Background, text, button, and border colours as HEX — applied as CSS custom properties, not hard-coded.

Do not add responsiveness yet. Do not add hover states yet. Colour only.
Layer 2 / 02

Typography

Your Google Fonts heading and body font. Sizes from your text style definitions — line-height and letter-spacing included.

Do not add font-weight unless it is in your design system. Match what Figma defines.
Layer 2 / 03

Spacing

The 8px grid scale — 8, 16, 24, 32, 48, 64px. Section padding, container max-width, and card gaps, all as CSS custom properties.

Do not add animations, transitions, or box shadows yet. Spacing only.

Not Hard-Coded Values.

Linking
Google Fonts

Copilot cannot guess a font it hasn't been told about. Link it, then name it — explicitly, every time.

1

Find your font

Go to fonts.google.com, find your heading font, and copy the embed <link> tag.

2

Order matters

The link tag goes in <head>, before the stylesheet link.

3

Reference it exactly

Use the exact font name in your CSS — quoted if it's multi-word.

4

Tell Copilot both names

Heading font and body font, explicitly, in the prompt. It cannot guess.

When it goes wrong

Common Layer 2
Failures

!

Font not loading

Add the Google Fonts link for [FONT NAME] to the head. Use the exact embed URL from fonts.google.com.

!

Wrong colour applied

Change the background-color of .hero to #[HEX]. It must match my design system exactly.

!

No max-width — content spans full screen

Add a .container class with max-width: 1200px and margin: 0 auto. Wrap all section content in it.

!

Cards stacking vertically

Apply display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px; to the .cards section.

!

Spacing too tight or too loose

Change all padding and margin values to multiples of 8px. Use my spacing scale: 8, 16, 24, 32, 48, 64.

Your design system exists to be used. Not once — everywhere.

A colour you pick once in Figma and then hard-code differently in CSS is not a design system. It is a coincidence.
